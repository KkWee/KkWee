Objective-C语法基础

OC使用“消息结构”，而非“函数调用”。两者之间的区别在于使用“消息结构”的语言，其运行时所执行的代码由运行环境所界所决定；而使用“函数调用”的语言，则由编译器所决定。假如调用的函数是多态的，那运行时要按照“虚方法表”来查出应该调用哪个函数。
OC无论是非多态，总是在运行时才会去查找所要执行的方法。

runtime component 运行期组件

OC的重要工作都由runtime component 而非编译器完成，全部的数据结构和函数都在运行期组件里面。
它含有所有的内存管理方法。运行期组件本质是一种与开发者所编写的代码相链接的“动态库”，其代码能把开发者编写的程序粘合起来。

对象和指针

OC语言中的指针是用来指示对象的。

someString指向分配在堆里的某块内存，其中含有一个NSString对象。对象所占用的内存空间只能分配在堆上。分配在堆上的内存必须直接管理。

属性

属性是OC的一项特性，用于封装对象中的数据。

在对象内部尽量直接访问实例变量

在对象之外访问实例变量，总是应该通过属性去做。而在对象内部访问实例变量时，应该采用直接访问的方式。

这两种写法有几个区别

  • 由于不经过OC的“方法派发”，所以直接访问实例变量的速度比较快。编译器产生的代码直接访问保存实例变量的内存
  • 直接访问，不会调用器设置方法。
  • 不会触发KVO
  • 通过属性来访问，有助于排查与之相关的错误。

还有一个问题是“惰性初始化” 也就是懒加载

ARC小节

  • assign: “设置方法”只会针对“纯量类型”（CGFloat， NSInteger等）进行简单的赋值操作。
  • strong: 这是一种“拥有关系”，为这种方法设置新值时，会先保留新值，再释放旧值，再将新值设置上去。
  • weak: 这是一种“非拥有关系”，设置方法既不保留新值，也不释放旧值。当属性所指的对象遭到摧毁时，属性值也会被清空
  • copy:与strong类似，但是并不保留新值，而是将其copy，经常用于NSString等类型的实例。
    如果将某个属性声明为copy，那么就应该在“设置方法”中拷贝相关对象，否则会误用该属性的使用者。

Strong指针
如果有一个UItextFiled，self.UItextFiled.text = @”KEWEN”;
NSString name = self.UItextFiled.text;
此时相当于有两个指针指向一个对象，而当self.UItextFiled.text = @”XU”后，
UItextFiled.text指针指向了新的对象，而name指针还是指向 @”KEWEN”对象。
只有当name变量获得新值,或者不再存在时(如局部变量方法返回时、实例变量对象释放时),原先的NSString 对象就不再拥有任何所有者,retain计数降为0,这时对象会被释放。
Weak指针
__weak NSString name = self.UItextFiled.text; 此时的name指针对@”KEWEN”对象并不是拥有关系，如果text发生改变，name会变成nil。
在UIView中，空间的属性设置也大多为weak，这样避免了野指针的产生，也避免了一些bug。在UI控件中，通常使用的是weak，

@property (nonatomic, weak) IBOutlet UIImageView *imageView;

在初始化这个对象时，不能用 self.imageView = [UIImageView alloc] init];
因为imageView对象是weak修饰的，只有一个对象指向weak对象时，会被系统释放，所以这里要
用到一个strong指针。

UIImageView *newImage = [UIImageView alloc] init];
self.imageView = newImage;
把这个strong指针赋值给weak指针。这样系统就不会自动释放。
